# 活动助手切换线上环境

> 原始资料：[[raw/活动助手切换线上环境|活动助手切换线上环境]]

## 概述

本文档描述从测试环境切换回生产（线上）环境的完整操作流程。这是 [[sources/活动助手测试环境安装]] 的反向操作。

## 操作步骤

### 1. 删除测试环境文件

删除活动助手安装目录中的 `.test` 文件：

- `zhanba/` 目录下的测试文件

![[Attachments/Pasted image 20260407200445.png]]

- `zhanba_ser/` 目录下的测试文件

![[Attachments/Pasted image 20260407200520.png]]

### 2. 恢复配置文件

修改 `zhanba/` 目录中的 `app.ini`：

- 注释掉（或删除）测试环境的 URL 配置
- 恢复生产环境配置

![[Attachments/Pasted image 20260407200716.png]]

### 3. 清理注册表

打开注册表编辑器，删除以下项：

- `zhanba_cli`
- `Yiletoo`

![[Attachments/Pasted image 20260407200929.png]]

### 4. 恢复 Hosts

在系统 hosts 文件中注释掉（或删除）测试环境的域名绑定。

![[Attachments/Pasted image 20260407201432.png]]

### 5. 启动程序

按 [[sources/活动助手测试环境安装]] 中的启动步骤启动活动助手。

## 对比：测试环境 ↔ 线上环境

| 步骤 | 切换到测试环境 | 切换到线上环境 |
|------|---------------|---------------|
| 配置文件 | 修改为测试 URL | 注释测试 URL，恢复默认 |
| 程序文件 | 复制为 `.test` 版本 | 删除 `.test` 文件 |
| Hosts | 添加测试域名绑定 | 注释测试域名绑定 |
| 注册表 | 清理以重新登录 | 清理以重新登录 |

## 注意事项

- 切换环境前确保已关闭所有活动助手相关进程
- 操作注册表前建议备份
- 切换后需要重新登录门店
