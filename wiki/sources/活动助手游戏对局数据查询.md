# 活动助手游戏对局数据查询

> 原始资料：[[raw/活动助手游戏对局数据查询|活动助手游戏对局数据查询]]

## 概述

本文档描述活动助手无畏契约赛事对局数据的查询方法和常见问题排查流程。

## 数据上报链路

```
活动助手客户端 → zhanba_expert_event → esports_match_data_team
      ↓                ↓                      ↓
   采集数据        原始事件表              加工后的对局数据
```

## 查询方法

### 第一步：检查原始数据上报

查询 `zhanba_expert_event` 表确认数据是否已上报：

**方式一：梧桐树自助查询**

```sql
-- 通过用户 uid 查询数据
select * from zhanba_expert_event 
where uid in (103667905, 103667940) 
  and timeline = '20260318'
```

> **获取 uid**：通过客户端赛事详情接口返回数据中的 `currentTeamPersons` 字段获取。

![[Attachments/Pasted image 20260407195637.png]]

**方式二：梧桐树我的数据**

在"我的数据"中模糊搜索 `zhanba_expert_event` 表，点击"数据详情"查看每天的数据。

![[Attachments/Pasted image 20260407195942.png]]

### 第二步：检查对局数据加工

确认数据是否推送到 `zhanba_expert` 数据库的 `esports_match_data_team` 表：

```sql
-- 通过赛事 id 查询数据
select * from esports_match_data_team 
where match_id = 3182
```

## 故障排查

### 游戏数据未成功上报的本地排查步骤

1. **检查文件版本** - 是否已替换活动助手最新文件
2. **检查 Hosts 绑定** - `C:\Windows\System32\drivers\etc\hosts` 是否已绑定测试环境 IP
3. **检查日志文件** - 查看 `zhanba` 安装目录 `log/` 中的日志：
   - 无畏契约赛事相关查看 `VALSdk*.log`

## 关键概念

| 概念 | 说明 |
|------|------|
| zhanba_expert_event | 活动助手专家模式事件表，存储原始上报数据 |
| esports_match_data_team | 赛事对局数据表，存储加工后的团队对局数据 |
| VALSdk | 无畏契约（Valorant）SDK，负责采集游戏数据 |
| timeline | 时间线标识，格式如 `20260318` |
| match_id | 赛事对局唯一标识 |
