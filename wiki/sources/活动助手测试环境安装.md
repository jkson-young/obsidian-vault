# 活动助手测试环境安装

> 原始资料：[[raw/活动助手测试环境安装|活动助手测试环境安装]]

## 概述

本文档描述活动助手客户端连接测试环境的完整配置流程。

## 核心步骤

### 1. 程序环境切换

需要准备三个程序包并修改配置文件：

| 程序 | 操作 |
|------|------|
| zhanba_ser | 复制 `zhanba_ser.exe` 并重命名为 `zhanba_ser.test` |
| zhanba | 复制 `NetBarInfo.exe` → `NetBarInfo.test`<br>复制 `ZhanBa.exe` → `NetBarInfo.test` |
| zhanba | 修改 `config\app.ini`，将 url 指向测试服务器 |

**配置截图：**
![[Attachments/d5dafc22-af26-4fa8-9a06-8fd2e487263c.png]]

### 2. 门店注册

在管理后台新建门店，使用 gid 作为门店名称。

![[Attachments/Pasted image 20260403105658.png]]

### 3. Host 绑定

在系统 hosts 文件中添加 DSSP 测试环境域名映射。

### 4. 启动顺序

1. 启动 `zhanba_ser_login.exe` 并登录

![[Attachments/Pasted image 20260403134855.png]]

2. 启动 `zhanba_cli.exe`（成功时服务端客户端在线数+1）

![[Attachments/Pasted image 20260403135157.png]]

3. 启动 `ZhanBa.exe` 打开主程序

## 关键概念

- **zhanba_ser**: 活动助手服务端程序
- **zhanba_cli**: 活动助手命令行客户端
- **DSSP**: 活动助手依赖的后端服务系统
- **gid**: 门店全局唯一标识
