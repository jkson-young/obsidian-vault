# 设备信息上报美团

## 流程概述

易乐途门店设备信息通过途邦SCRM上报美团平台的完整链路。

## 架构图

```
┌─────────────────┐
│   易乐途门店     │
│   (设备数据源)   │
└────────┬────────┘
         │ 设备开关机/数据变更时上报
         ↓
┌─────────────────┐     ┌─────────────────┐
│   途邦 SCRM     │────→│   数据清洗规则   │
│  (数据中间层)    │     │  (过滤无效数据)  │
└────────┬────────┘     └─────────────────┘
         │ 清洗后数据存储
         ↓
┌─────────────────┐
│    数据存储层    │
│  (mt_shop_auths  │
│   等数据表)      │
└────────┬────────┘
         │ 美团主动查询时
         ↓
┌─────────────────┐
│    美团平台      │
│  (数据消费方)    │
└─────────────────┘
```

## 关键特性

### 被动推送

> **重要**: 途邦不会主动向美团推送数据。只有当美团调用查询接口时，数据才会同步给美团。

### 数据触发时机

| 触发方式 | 说明 |
|---------|------|
| 设备自动上报 | 设备开机、关机、重启时自动上报 |
| 人工修改 | 通过"编辑设备信息"修改后触发同步 |

### 绑定关系

门店必须先完成授权绑定，数据才能同步：

```
易乐途门店 ←──绑定──→ 美团门店
   │                       │
   └─── 设备数据 ───────────┘
```

## 数据内容

设备全量信息包括：

| 类别 | 字段示例 |
|------|---------|
| 基础信息 | name, uuid, gid, shop_id |
| 处理器 | cpu (型号、核心数、频率) |
| 显卡 | gpu (型号、显存) |
| 内存 | memories (容量、频率) |
| 存储 | disks (型号、容量) |
| 显示器 | monitor (尺寸、分辨率) |
| 网络 | mac, ip, netcard |
| 外设 | hids (键盘、鼠标) |
| 系统 | system_name, system_version |
