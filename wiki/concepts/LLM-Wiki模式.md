# LLM-Wiki模式

## 定义

LLM-Wiki 是一种使用大语言模型（LLM）增量构建和维护个人知识库的模式。区别于传统的 RAG（检索增强生成），它强调**知识的持久化积累和复利效应**。

## 核心特征

| 特征 | 说明 |
|------|------|
| **持久化** | Wiki 是长期存在的产物，不是每次查询临时生成 |
| **增量构建** | 每份新资料都整合进现有知识体系 |
| **交叉引用** | 页面间相互链接，形成知识网络 |
| **主动维护** | LLM 负责更新、修正、一致性检查 |
| **人机协作** | 人负责策划和提问，LLM 负责执行和维护 |

## 三层架构

```
┌─────────────────┐
│  Raw Sources    │  ← 原始资料（只读）
│  （用户放入）    │
└────────┬────────┘
         │ Ingest
         ↓
┌─────────────────┐
│  The Wiki       │  ← 知识库（LLM维护）
│  （LLM生成）     │
└────────┬────────┘
         │ Query/Lint
         ↓
┌─────────────────┐
│  The Schema     │  ← 模式定义（人机共创）
│  （CLAUDE.md）   │
└─────────────────┘
```

## 三种操作

### Ingest（摄入）
将新资料整合进 wiki：
- 读取原始资料
- 提取关键信息
- 创建/更新 wiki 页面
- 维护交叉引用

### Query（查询）
基于 wiki 回答问题：
- 搜索相关页面
- 综合信息
- 可归档有价值的新发现

### Lint（健康检查）
定期检查 wiki 质量：
- 发现矛盾
- 标记陈旧内容
- 识别孤立页面
- 填补缺失链接

## 与 RAG 的对比

| 维度 | RAG | LLM-Wiki |
|------|-----|----------|
| 知识存储 | 原始文档 | 结构化 wiki |
| 检索方式 | 向量检索 | 索引+链接浏览 |
| 知识积累 | 无 | 持续复利 |
| 一致性 | 每次重新推导 | 已维护好 |
| 人机分工 | 人查询，LLM检索生成 | 人策划提问，LLM维护 |
